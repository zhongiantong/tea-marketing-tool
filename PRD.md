# 「茶言觀色」精準行銷與流失預警系統 - 產品需求文件 (PRD)

## 1. 產品概述
本工具為 **Teasight Dashboard**，旨在透過顧客的「到店天數」、「APP 使用狀態」及「優惠券歷史」等行為特徵，即時進行客群分群（Segmentation）並自動產出差異化行銷決策建議。

---

## 2. 核心數據邏輯 (Based on K-Means Analysis)

### 2.1 客群分群定義 (Segment Profile)
根據機器學習模型訓練結果，系統將顧客動態劃分為以下三個核心群體：

| 群體編號 | 內部代碼 | 描述 | 流失率 | 數據特徵 (Cluster Stats) |
| :--- | :--- | :--- | :--- | :--- |
| **群體 2** | `C2` | **高貢獻顧客** | **0.0%** | **VIP 級別**：月均到訪 30 次，半年總消費額 $108,000。極度忠誠。 |
| **群體 1** | `C1` | **數位型顧客** | **18.0%** | **主力客群**：APP 活躍用戶，月均到訪約 7.9 次，半年消費約 $3,588。 |
| **群體 0** | `C0` | **傳統顧客** | **37.0%** | **高危邊緣**：**完全不使用 APP**，仰賴實體接觸，流失風險最高。 |

---

## 3. 交互與決策邏輯 (Interaction & Decision Logic)

### 3.1 策略觸發機制 (Smart Ticking)
系統根據動態參數即時運算，於策略卡片標註「建議執行 (Tick)」標籤：

#### A. 優先優惠券 (折扣)
- **C2 高貢獻顧客**：常駐顯示 (**Cyan 星號**)，定義為「專屬 VIP 禮遇」。
- **C0/C1 群體**：僅在 **天數 > 15d** 時顯示，作為喚回誘因，防止流失。

#### B. 口味樣品 (產品)
- **C2 高貢獻顧客**：常駐顯示 (**Cyan 星號**)，定義為「新品優先品鑑」。
- **C1 數位型顧客**：常駐顯示 (**Green 剔號**)，維持品牌新鮮感。
- **C0 傳統顧客**：僅在 **天數 > 7d** 時顯示，增加到店誘因。

#### C. SMS 提醒 (渠道)
- **C0 傳統顧客**：由於無 APP 溝通渠道，SMS 為主要觸達手段，判定門檻較低。
- **通用邏輯**：於 **天數 > 30d** 時強制亮起 (**Red 剔號**)，觸發緊急喚回流程。

---

### 3.2 流失風險狀態判定
- **穩定狀態**：當「到訪天數」$\le 30$ 且具備 APP 活躍度時。
- **⚠️ 高流失模式**：當天數 $> 30$ **或** 用戶為 **群體 0 (無 APP)** 時，系統全域 UI 切換至霓虹紅警示模式。

---

## 4. UI/UX 視覺規範 (Frontend Specs)
- **色調系統**：
  - 基調：`#0C0C0C` (Linear Dark Mode)
  - 活躍/安全：`#8EFF33` (Neon Green)
  - 高貢獻/VIP：`#22D3EE` (Cyan)
  - 流失警示：`#FF3333` (Neon Red)
- **動畫規範**：
  - **Progress Streak**：進度條流光效果，代表 AI 模型實時運算中。
  - **Glow Effect**：高風險數值需具備外發光效果以引起管理者注意。

---

## 5. 技術實現 (Technical Implementation)
- **核心框架**: Vanilla JavaScript (ES6+) + Tailwind CSS 3.0。
- **數據綁定**: 透過 `updateDisplay()` 函式實現數據雙向映射。
- **分群演算法映射**: 將 Python 端處理之 Cluster 特徵轉化為前端 JS Object 邏輯判定。
